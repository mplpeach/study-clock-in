# 🌱 学习打卡网站

一个功能完整的学习打卡网站，支持目标管理、任务规划、实时/补录打卡、图片上传、日历视图和统计分析。

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Java 17 + Spring Boot 3.2 + Spring Data JPA + MySQL 8 |
| 前端 | React 18 + TypeScript + Ant Design 5 + ECharts |
| 构建 | Maven + Vite |
| 部署 | Docker / docker-compose |
| 图片存储 | 本地文件系统（可扩展为 MinIO / 阿里云 OSS） |

## 快速启动

### 方式一：本地开发

**前置条件：** JDK 17+、Node.js 20+、MySQL 8.0+

```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS study_clock_in DEFAULT CHARSET utf8mb4;"

# 2. 启动后端
cd backend
mvn spring-boot:run

# 3. 启动前端（新开终端）
cd frontend
npm install
npm run dev
```

访问 http://localhost:5173 即可使用。

### 方式二：Docker 部署

```bash
docker-compose up -d
```

访问 http://localhost 即可使用。

### 默认账号

系统自动创建默认用户，无需登录即可使用。

## 功能说明

### 🎯 目标管理

- 创建学习目标（如「算法学习」、「英语提升」）
- 每个目标可设置颜色标识，便于区分
- 目标下可关联多个可复用的学习任务

### 📋 任务管理

- 创建可复用的学习任务（如「刷 LeetCode」、「背单词」）
- 一个任务可关联到多个目标
- 支持关键词搜索任务
- 日历视图展示每日任务

### ✅ 打卡系统

支持三种打卡模式，覆盖完整学习闭环：

**实时打卡**

```
[开始学习] → 计时器启动 → [暂停] / [结束学习]
                  ↑              │
                  │   记录内容，保持进度     │  记录内容+心得+图片
                  │   可切换任务            │  任务标记为已完成
                  │              │
                  └── [继续学习] ←┘
                    （计时从累计时间恢复）
```

- 开始学习时支持选择已有任务或一键新建
- 学习进行中，正上方显示计时器卡片，[暂停] 和 [结束学习] 两个操作按钮
- **暂停**：停止计时，弹窗记录当前学习内容，任务保持「进行中」状态，可切换去学别的任务
- **继续学习**：从暂停的实例恢复，计时器从累计时间继续（非从零开始），多次暂停-继续计时精确累计
- **学习内容回显**：再次暂停/结束学习时，弹窗自动回显上一次的学习内容，方便追加修改
- **结束学习**：填写学习内容 + 心得 + 上传图片，任务标记为「已完成」
- 今日任务列表呈现三种状态：待开始 → [开始学习]、进行中 → [继续学习]、已完成 → ✅ 标签；正在活跃计时的任务不显示操作按钮，展示实时计时

**补录打卡**

- 选择已有任务或新建 → 选择时间段 + 时长 → 填写内容和心得 → 上传图片
- 适用于学习结束后补记的场景

| 模式 | 适用场景 |
|------|----------|
| **实时打卡 + 暂停/继续** | 在电脑前学习，中途需要切换任务或休息 |
| **实时打卡 + 直接结束** | 一次性完成学习 |
| **补录打卡** | 学习结束后补记 |

### 📸 学习笔记

- 每次打卡记录学习内容和心得
- 支持上传图片
- 图片以打卡记录为单位聚合展示

### 📅 日历视图

- 按月查看每日任务分布
- 逾期未完成的任务红色高亮警示
- 点击日期查看该日所有任务详情

### 📊 统计可视化

- **打卡日历热力图**：类似 GitHub 绿格子，展示每天学习时长
- **连续打卡天数**：当前连续天数 + 历史最长连续天数
- **累计统计**：总打卡天数、总学习时长
- **趋势图**：每日学习时长柱状图
- **目标分布**：各目标学习时长饼图
- **任务完成进度**：各目标下的任务完成率

## 数据模型

```
用户 (User) ──→ 目标 (Goal) ──←→── 任务定义 (Task)
                    │                       │
                    │                 每日任务实例 (TaskInstance)
                    │                 状态：待开始 → 进行中 → 已完成
                    │                       │
                    └──────────┬────────────┘
                               │
                         打卡记录 (CheckInRecord)
                               │
                         打卡图片 (CheckInImage)
```

核心实体关系：

| 实体 | 说明 |
|------|------|
| **User** | 用户（已预留，初期写死默认用户） |
| **Goal** | 学习目标，如「算法学习」 |
| **Task** | 可复用的任务定义，与目标多对多 |
| **GoalTask** | 目标-任务关联表 |
| **TaskInstance** | 每日任务实例，状态：TODO → IN_PROGRESS → COMPLETED；暂停时保持 IN_PROGRESS，不流转 |
| **CheckInRecord** | 学习打卡记录（实时/补录），一个实例可有多条记录（每次暂停/继续产生新记录），通过 startTime/endTime 精确累计时长 |
| **CheckInImage** | 打卡图片，一对多关联打卡记录 |

## RESTful API 接口

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| **目标** | GET | `/api/goals` | 获取所有目标 |
| | POST | `/api/goals` | 创建目标 |
| | PUT | `/api/goals/{id}` | 更新目标 |
| | DELETE | `/api/goals/{id}` | 删除目标 |
| **任务** | GET | `/api/tasks` | 获取所有任务 |
| | GET | `/api/tasks/search?keyword=` | 搜索任务 |
| | POST | `/api/tasks` | 创建任务 |
| | PUT | `/api/tasks/{id}` | 更新任务 |
| | DELETE | `/api/tasks/{id}` | 删除任务 |
| | POST | `/api/tasks/bind` | 关联任务到目标 |
| | POST | `/api/tasks/unbind` | 取消关联 |
| | GET | `/api/tasks/by-goal/{goalId}` | 获取目标下任务 |
| **任务实例** | POST | `/api/instances` | 创建每日任务实例 |
| | POST | `/api/instances/batch` | 批量创建 |
| | GET | `/api/instances/date?date=` | 按日期查询 |
| | GET | `/api/instances/calendar?year=&month=` | 按月查询 |
| | GET | `/api/instances/overdue` | 获取逾期任务 |
| | DELETE | `/api/instances/{id}` | 删除实例 |
| **打卡** | POST | `/api/checkins/start` | 开始学习（实时模式） |
| | POST | `/api/checkins/end` | 结束/暂停学习（complete=true 完成实例，false 暂停保持进度；durationSeconds 精确计时） |
| | POST | `/api/checkins/manual` | 补录打卡（含图片） |
| | GET | `/api/checkins` | 获取所有打卡记录 |
| | GET | `/api/checkins/{id}` | 获取单条记录 |
| | GET | `/api/checkins/by-instance/{id}` | 按实例查询记录 |
| **统计** | GET | `/api/statistics` | 获取统计数据 |

## 后续可拓展方向

### 1. 用户系统 🔐
- **现状**：User 表已预留，写死默认用户
- **后续**：接入 Spring Security + JWT，实现注册/登录/权限管理
- **难度**：⭐（中等，方案成熟）
- **涉及修改**：新增 `AuthController`、JWT 工具类、Security 配置；所有 Controller 从写死 userId 改为从 Token 获取

### 2. 图片存储上云 ☁️
- **现状**：本地文件系统存储
- **后续**：实现 MinIO 或阿里云 OSS 的 `FileStorageService`，通过配置切换 `file.storage.type`
- **难度**：⭐（简单，接口已抽象）

### 3. 社交与排行榜 👥
- 好友系统：关注/粉丝、学习动态 feed
- 学习小组/自习室：组队打卡、小组学习时长排行
- 排行榜：日榜/周榜/总榜
- **难度**：⭐⭐⭐（涉及好友关系链、feed 流设计）

### 4. 任务模板与重复规则 🔄
- **现状**：手动创建每日任务实例
- **后续**：支持「每周一三五自动创建任务」「每周末重复」
- **难度**：⭐⭐（需要定时任务 + cron 表达式解析）

### 5. 学习提醒与推送 🔔
- 每日学习提醒（邮件/站内信）
- 连续打卡激励通知
- 逾期任务催办
- **难度**：⭐⭐（需要集成邮件或消息推送服务）

### 6. 移动端适配 📱
- **现状**：PC 端 Ant Design 布局
- **后续**：响应式适配或开发小程序
- **难度**：⭐⭐（响应式改造） / ⭐⭐⭐（小程序开发）

### 7. 数据导出 📊
- 打卡记录导出为 Excel/PDF
- 学习报告自动生成（周报/月报）
- **难度**：⭐（EasyExcel / JasperReports）

### 8. 更多统计维度 📈
- 学习时间段分析（什么时间段效率最高）
- 各任务耗时排名
- 目标完成预测（基于历史数据）
- **难度**：⭐⭐

### 9. 搜索与筛选 🔍
- 打卡记录全文搜索
- 多条件筛选（目标/任务/日期范围/状态）
- **难度**：⭐

### 10. 性能优化与运维 🚀
- Redis 缓存热门数据（目标列表、统计结果）
- 数据库读写分离
- CI/CD 流水线（GitHub Actions 自动构建部署）
- Nginx 反向代理与 HTTPS 配置
- **难度**：⭐⭐⭐

## 项目亮点（面试可用）

1. **接口隔离原则**：文件存储服务抽象为接口，可无缝切换实现
2. **合理的表设计**：任务定义与任务实例分离，支持任务复用与每日独立状态
3. **清晰的 RESTful API**：规范的路由设计和响应格式
4. **统一异常处理**：全局异常拦截 + 统一响应体
5. **前后端分离**：React + Spring Boot 独立部署，Nginx 反向代理
6. **容器化部署**：Docker Compose 一键启动
7. **完整的文件上传流程**：从存储设计到前端图片预览
